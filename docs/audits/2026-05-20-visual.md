## 2. Artefacts détectés

### Erreurs console (toutes pages, en local uniquement)

```
Failed to load resource: the server responded with a status of 404 (Not Found)
  /_vercel/insights/script.js
  /_vercel/speed-insights/script.js
```

**Cause** : les SDK `@vercel/analytics` et `@vercel/speed-insights` tentent de charger leur script depuis un endpoint `/_vercel/...` qui n'existe **que sur Vercel deployment**. En `npm run start` local, c'est 404, mais sans impact fonctionnel (les libs gèrent silencieusement).

**Verdict** : pas un bug, artefact local. Le score Lighthouse "Best Practices 74" (vu en wave 1) venait probablement de là — sera plus propre en prod.

**Action** : aucune. Documenter dans le runbook observabilité si on relance Lighthouse en local.

### Mobile landing — count-up captured mi-animation

Le composant `CountUpOnReveal` lance une animation 0 → valeur finale au scroll-into-view. Les outils de capture automatisée (Playwright avec `waitUntil: networkidle`) peuvent screenshot avant la fin de l'animation, donnant un nombre intermédiaire (ex : 440 au lieu de 462).

**Verdict** : pas un bug user-facing (l'animation se termine en ~1s côté visiteur réel). C'est juste un artefact d'audit automatisé.

**Action** : aucune. Si on veut un screenshot stable pour comm/presse, ajouter un `page.waitForTimeout(1500)` avant le screenshot.
