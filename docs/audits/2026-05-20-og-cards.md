# Audit visuel — Pilier 1 : OG cards / social previews — 2026-05-20

**Scope** : audit des **OpenGraph cards** que Twitter/LinkedIn/Slack affichent en preview quand quelqu'un partage l'URL d'une page. C'est la **première impression** d'un visiteur amené par un partage social — déterminant pour le taux de clic.

**Méthode** : Playwright fetch de 20 pages clés → extraction des balises `og:title/description/image` → vérification que l'image OG render (HTTP 200) → screenshot de chaque OG image pour inspection visuelle.

`/methode` et `/corrections` — pages que tu partagerais pour démontrer la rigueur du projet — n'ont **aucune OG image non plus**.

## 1. OG cards existantes — qualité

### ✅ Landing
**Design** : Brand FO + headline citoyen + 3 KPI (Budget, Marchés, Subventions) + source

⚠️ **Pas aligné avec notre fix lede temporel** :
- Affiche "BUDGET 2026" sans préciser "VOTÉ"
- Affiche "23 000 marchés" sans "depuis 2013"
- Affiche "46 000 subventions" sans "depuis 2018"

→ Le lede de la page est honnête, l'OG card ne l'est pas. Mismatch.

### ✅ /ville/paris/subventions — exemple à dupliquer
**Design** : kicker `/subventions` + lede "Qui reçoit l'argent public ?" + Montant 1,35 Md€ + 5977 subventions + sources

Excellent — c'est exactement le template à appliquer aux pages sans OG.

### ✅ /ville/paris/logement
**Design** : "2657 logements produits face à 195828 demandes" + 3 KPI + sources

⚠️ **Bug visuel** : "195828demandes" sans espace entre le nombre et le mot. Minuscule mais visible.

**Design** : "Ce qu'on paie : la structure des prélèvements" + 43,4% du PIB + 1 268 Mds€

⚠️ **Bug visuel** : carré vide `□` à côté de "1 268 Mds€" — probablement un caractère unicode (espace insécable ou €) non rendu. À fixer.

### ✅ /france/daily-bread — l'OG le plus impactant
**Design** : Format square (1080×1080, parfait Instagram/LinkedIn). Headline "Mon mois de service public" + 4 équivalents concrets + stack bar Sécu/État/Local + persona.

⚠️ **Petit bug data** : "≈ 0.0 jours d'école pour 1 élève public" — la valeur est à 0, soit le calcul est cassé pour cet équivalent.

### ✅ Autres OG (analyses, marches, invest, dette, fr-dette, fr-etat)
Toutes propres et cohérentes avec le design system. Bonnes.

## 2. OG cards manquantes (7 pages critiques)

### 🚨 /ville/paris/budget — la PAGE PRINCIPALE
Aucune `og:image`. Quand quelqu'un partage `/ville/paris/budget`, Twitter affiche juste le texte. **Impact maximal sur taux de clic.**

**Fix recommandé** : créer `website/src/app/ville/paris/budget/opengraph-image.tsx` qui affiche :
- Headline "Le budget de Paris"
- Montant total (Md€) avec libellé "voté" ou "exécuté" selon l'année
- 2 KPI : Fonctionnement (%) / Investissement (%)
- Sources : Comptes administratifs M57

### 🚨 /methode
Page de référence pour journalistes/décideurs. Sans OG card, partager `/methode` donne un preview vide.

**Fix recommandé** : OG simple avec "Méthode" + tagline "5 étapes : Source → Staging → Core → Mart → Site" + sources clés (OpenData Paris, INSEE, Eurostat).

### 🚨 /corrections
Page transparence — c'est celle qu'on pousse pour répondre à "et si vous vous trompiez ?". Sans OG, pas de visuel pour démontrer la rigueur.

**Fix recommandé** : OG avec "Corrections publiques" + nombre de corrections + dernière en date.

### 🚨 /contact
**Fix recommandé** : OG simple avec headline contact + email visible.

### ⚠️ /ville/marseille/budget + /ville/marseille/subventions
Mêmes templates que les versions Paris, juste avec data Marseille. Le bandeau WIP ne devrait PAS apparaître dans l'OG (utiliser le format propre Paris).

### Moins critique : /accessibilite, /confidentialite, /mentions-legales, /licence
Légales, on partage rarement. Un OG générique (fallback root) suffirait — mais elles n'en ont pas non plus actuellement.

## 3. Drill pages — couverture mixte

Pages drill qui ONT un OG : `subventions/association/[slug]`, `marches/contrat/[numero]`, `marches/fournisseur/[siren]`, `investissements/projet/[id]`.

Pages drill SANS OG (auditées en passant) :
- `subventions/theme/[slug]`
- `marches/categorie/[slug]`
- `investissements/chapitre/[slug]`
- `investissements/arrondissement/[num]`
- `logement/arrondissement/[arr]`
- `dette/bailleur/[slug]`

Moins critique que les pages principales mais utile si des liens drill sont partagés (ex : fiche thème "Logement" qui pourrait faire un thread Twitter).

---

## 4. Pourquoi le fallback root ne marche pas

Next.js `opengraph-image.tsx` n'est PAS hérité automatiquement par les sous-routes. Chaque route a besoin de son propre fichier OU d'un layout parent qui définit `metadata.openGraph.images`.

Actuellement la convention du projet : un `opengraph-image.tsx` par route principale. Les pages sans fichier dédié n'ont **aucune** OG image (pas de fallback automatique vers le root).

**Cohérent avec les bonnes pratiques** (OG personnalisé par page > fallback générique), mais demande de la maintenance.
