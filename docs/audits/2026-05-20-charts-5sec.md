# Audit visuel — Pilier 2 : Charts shareable + 5-sec test — 2026-05-20

**Scope** : audit qualité visuelle des charts (Sankey, treemap, choropleth, bars, timelines) + test de la **première impression** sur les pages principales (le "5-second test" : qu'est-ce qu'un visiteur comprend en 5 secondes ?).

**Méthode** : screenshots Playwright (fullpage + above-the-fold) sur 12 pages clés en desktop 1440×900, analyse visuelle.

---

## Synthèse

✅ **Storytelling éditorial très solide partout** — toutes les pages principales suivent le même pattern : kicker numéroté → H1 court avec italique → lede court avec sources cliquables → year picker → hero text storytelling avec chiffres saillants → graphes → drill cards. Cohérence forte.

✅ **5-sec test passe sur 11/12 pages** — un visiteur comprend immédiatement de quoi parle la page et trouve le chiffre principal.

✅ **Hiérarchie visuelle propre** : kicker → H1 → lede → CTA → chart. Le regard va naturellement vers le bon endroit.

⚠️ **1 finding critique sur /france/budget** : hauteur de page 18329px (vs 4-6k normal sur les autres pages), avec un **énorme blanc** entre les sections. Probable bug rendering chart ou container hauteur fixe.

⚠️ **Charts globalement OK mais quelques détails à polir** (cf section 4).

## 4. Findings consolidés

### 🚨 Critique
1. **`/france/budget` page 18329px avec massive blank space** entre recettes-apu (~3900px) et drill content (~5000px). Suggère un container hauteur fixe ou un chart qui ne render pas. À investiguer (probablement un composant drill explorer qui calcule une hauteur basée sur N enfants × hauteur fixe au lieu de hauteur réelle).

### ⚠️ Polish
2. **Charts Sankey sur mobile** : possible scroll horizontal pénible. Tester explicitement sur viewport mobile.
3. **Choropleth Leaflet sur mobile** : interactions complexes. Vérifier qu'on peut zoom-out facilement.
4. **/france/daily-bread bucket "education"** : était hardcoded sur "education" mais le vrai key est "education_recherche" — FIXÉ dans PR #50. Pareillement, vérifier que d'autres `state_breakdown` keys sont bien matchés ailleurs.

### ✅ Bons points (à valoriser)
5. **Storytelling éditorial cohérent** sur toutes les pages : kicker → H1 → lede → year picker → hero text → charts → drill cards → "À lire". C'est exactement le format Le Monde Décodeurs / Bloomberg / FT.
6. **Hero text en bleu cliquable** : chaque chiffre est cliquable et mène à un drill ou tooltip. Excellent UX.
7. **Year picker partout** : avec couleur rouge sur voté + libellé "voté"/"exécuté" — transparence remarquable.
8. **Tooltips sources** : chaque mot technique (SRU, OPH, M57, comptes administratifs) a un Tip avec définition. Très accessible.
